# techContext — Стек, инструменты, запуск

## Стек

| Слой | Технология | Почему |
|------|-----------|--------|
| Сборка/дев-сервер | **Vite** | HMR, быстрый билд, стандарт для фронтенд-игр |
| Язык | **TypeScript** (strict) | типы критичны, когда клиент и сервер делят код |
| 3D | **three.js** (`three/webgpu`) | WebGPURenderer + откат на WebGL2, TSL-материалы |
| ECS | **miniplex** | эргономичный TS, компоненты-объекты, лёгкая сериализация |
| Тесты | **Vitest** | нативно к Vite; чистая логика симуляции без GPU |
| Качество | **ESLint + Prettier** | единый стиль, ловля ошибок |
| Git | репозиторий инициализируется (`git init`) | версии, ветки, коммит спеки |

## Запуск (после скаффолдинга — этап 1)

- `npm install` — зависимости.
- `npm run dev` — дев-сервер с горячей перезагрузкой (заменяет «двойной клик по файлу»).
- `npm run build` — прод-сборка.
- `npm test` — юнит-тесты симуляции.

## Ограничения и внешние зависимости

- **Слой подписей грузится из сети** (Esri `World_Boundaries_and_Places`, CORS `*`). Спутниковый
  слой `World_Imagery` **удалён** (фича материала: планета — стилизованная биом-карта, а не фото).
  Базовый глобус — процедурная биом-текстура (офлайн); подписи городов — сетевой слой (без сети
  просто нет подписей). Landmask реальных материков запечён в код (`sim/landmask.data.ts`) — офлайн.
- **WebGPU** есть не везде — поэтому обязателен автоматический откат на **WebGL2**.
- Целевая платформа — **десктопный браузер**. Мобильный (pinch-zoom, тач) — вне текущего скоупа.

## Данные

- База ~220 крупнейших агломераций мира (название, широта, долгота, население) встроена в код —
  работает офлайн. Живёт в `sim/cities.ts`.
