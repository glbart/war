# Банк памяти проекта

> **ЯЗЫК ОБЩЕНИЯ — РУССКИЙ.** Всё общение с пользователем — на русском. Исключения только для
> технических терминов, названий библиотек, кода и команд. Правило действует во всех сессиях.

Этот каталог — источник истины по проекту. Он существует, потому что между сессиями память
обнуляется: чтобы продолжить работу, нужно прочитать эти файлы.

## Порядок чтения

1. **projectbrief.md** — что делаем и зачем (корневой документ, задаёт цель).
2. **productContext.md** — продукт: проблема, ощущения от игры, UX.
3. **systemPatterns.md** — архитектура и ключевые технические решения.
4. **techContext.md** — стек, инструменты, запуск, ограничения.
5. **activeContext.md** — текущий фокус, открытые вопросы, следующие шаги.
6. **progress.md** — что уже работает, что осталось, статус.

`activeContext.md` и `progress.md` меняются чаще всего — их держим актуальными.

## Как обновлять

После значимых изменений в проекте обновляй `activeContext.md` и `progress.md`.
Архитектурные решения фиксируй в `systemPatterns.md`. Смена стека — в `techContext.md`.
