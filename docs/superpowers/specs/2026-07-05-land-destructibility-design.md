# Физическая разрушаемость суши — дизайн

_Дата: 2026-07-05 · Ветка: `feat/land-destructibility` · Статус: автономно утверждён (пользователь делегировал решения)_

## 1. Проблема и цель

Удар по суше сейчас выглядит как **плоское чёрное пятно**: `DamageField` пишет канал гари
(G → почти чёрный `0.06,0.05,0.05` в `GlobeView.colorNode`) поверх биома + вдавливает чашу
(канал R → сдвиг вершины внутрь). Читается как «клякса», а не как физическое разрушение материала.

**Цель:** кратер должен читаться как **физически разрушенная поверхность**:
- **Морфология:** чаша + **приподнятый вал** (rim выше окружения) + **эжекта-покрывало** вокруг
  (выброшенный грунт, спад плотности к периферии).
- **Зоны материала** вместо однородной черноты: оплавленное стекло/спёкшийся грунт в центре →
  обнажённая порода/обугленные стенки → выброс (пыль/обломки) на кольце → **гарь-градиент**,
  плавно затухающий в биом. Не слэб near-black.
- **Рельеф читается на свету:** микродеталь нормали в зоне кратера (скальная шероховатость),
  чтобы статичное солнце давало тени вала/стенок.
- **Выброс материала:** баллистические **частицы эжекты** (грунт/обломки/пыль) при детонации —
  взлетают и падают обратно.

### Граница скоупа (YAGNI)

- **Только визуал.** Геймплей (жертвы/урон) не меняется (как в фиче материала и в воде).
- **Не делаем:** настоящую физику/коллизии обломков; фрактурную геометрию (раскол меша на куски);
  постоянные кучи обломков как отдельная геометрия; воронку под водой (там своё, подпроект №1);
  LOD/детализацию суши в целом (это подпроект 2B — отдельная спека). Микродеталь нормали здесь —
  **локально в зоне кратера**, не глобальный detail-слой.
- Соответствие реальной баллистике эжекты — **стилизованное** (форма r^-2.7 как ориентир, не физрасчёт).

## 2. Опорная архитектура (что переиспользуем)

- `render/DamageField.ts` — накопительное equirect-поле (ping-pong max, `cos(lat)`-поправка,
  splat штампом). Сейчас каналы: **R**=глубина чаши, **G**=гарь, **B**=оплавление/полынья (лёд).
  Канал **A** свободен.
- `render/GlobeView.ts` — `positionNode` (вдавливание по R), `colorNode` (биом→гарь→полынья),
  `bumpMap`=топо-текстура. Глобус — `MeshPhongNodeMaterial` (есть статичный свет сцены).
- `render/effects/particles.ts` — `ParticlePool`: инстанс-спрайты, движение в TSL из атрибутов
  aA/aB/aC, кольцевой буфер, один `uTime`, ноль аллокаций/кадр. **Кривая высоты сейчас
  монотонный подъём** (`1-(1-lk)^2`) — для дыма/огня; для эжекты нужна **баллистика**
  (взлёт+падение, парабола).
- `render/Scene.ts` — `startExplosion` ветка суша/лёд (`else`): `explosionView.spawn` +
  `particlePool.emit` + `decalView.spawn` + `damageField.splat(dir, yield, kind)`.

## 3. Согласованные решения (автономно)

| Вопрос | Решение |
|--------|---------|
| Морфология кратера | Чаша + приподнятый вал + эжекта-апрон, **запекается в профиль штампа `DamageField`** |
| Кодирование поля | Переосмыслить каналы: **R**=глубина чаши (вниз), **A**=высота вала+эжекта (вверх), **G**=гарь/материал-микс, **B**=оплавление (лёд, без изменений). Сдвиг вершины = `−N·R·DEPTH + N·A·RIM` |
| Зоны материала | Радиальные пояса по профилю (глубина R + кольцо A): стекло-центр → порода-стенки → выброс-кольцо → гарь-градиент → биом. Гарь — **градиент**, не слэб |
| Рельеф | Процедурная микродеталь нормали в damaged-зоне (шум по позиции), сила по (R+A). Через `normalNode`/возмущение или усиление bump локально |
| Эжекта | Новые **баллистические GPU-частицы** (грунт/пыль). Переиспользовать `ParticleMesh`-паттерн, но кривая высоты — параболическая (взлёт+падение). Отдельный пул или обобщение `particles.ts` |
| Накопление | Кратеры сливаются как сейчас (max по каналам) — новые каналы тоже max |
| Где живёт | Только render; sim/геймплей не трогаем |

## 4. Архитектура и компоненты

Шов `Command → Simulation → Events → Renderer` без изменений. Всё — в render, на существующем
событии `explosionStarted` (суша/лёд ветка).

**`render/DamageField.ts` (модификация).**
- Профиль штампа: помимо чаши `bowl`, добавить **вал** (кольцевой бугор чуть за краем чаши) и
  **эжекта-апрон** (спадающее к периферии наслоение). Записать высоту-вверх в канал **A**.
- Гарь (G) переосмыслить как «интенсивность материала-микса» с более мягким радиальным спадом
  (градиент до 0), чтобы не было резкого чёрного края.
- Каналы задокументировать в шапке файла (R/G/B/A).
- Слияние — по-канальный `max` с prev (уже есть), включая A.

**`render/GlobeView.ts` (модификация).**
- `positionNode`: `positionLocal − normalLocal·(R·MAX_CRATER_DEPTH) + normalLocal·(A·RIM_HEIGHT)`.
- `colorNode`: заменить одноцветную гарь на **радиальные зоны материала** по (R, A, G):
  оплавленное стекло (тёмное, глянцевое-низкосатурированное) в центре (высокий R) →
  обнажённая порода/обугленные стенки (серо-бурый) на склоне → выброс/пыль (светлее биома) на
  кольце вала (A) → **гарь-градиент** (потемнение с плавным затуханием) → биом. Лёд-полынья (B)
  сохраняется.
- Микродеталь: возмущение нормали процедурным шумом в зоне `(R+A)>0`, сила ∝ (R+A) — рельеф вала
  и стенок ловит статичный свет. (Реализация: `normalNode` или локальное усиление `bumpScale` —
  уточнить в плане по API three 0.185.)

**`render/EjectaView.ts` (новый) ИЛИ обобщение `particles.ts`.**
- Баллистические частицы выброса: спавн из эпицентра, разлёт наружу+вверх, **падение под
  «гравитацией»** (парабола `h = v0·τ − ½·g·τ²`), гаснут у поверхности. Цвет — грунт/пыль
  (бурый→серый), доля тёмных «обломков». Инстанс-спрайты, TSL-движение из атрибутов, кольцевой
  буфер, ноль аллокаций/кадр — как `ParticlePool`.
- Решение форму (отдельный класс vs добавить «ballistic»-режим в `ParticleMesh`) — в плане;
  предпочтение: **отдельный `EjectaView`** (не трогать рабочий меш гриба), переиспользуя приёмы.

**`render/Scene.ts` (модификация).**
- В `startExplosion` ветке суша/лёд: добавить `ejectaView.emit(dir, yield, seed, clock)` рядом с
  существующими `particlePool.emit`/`decalView.spawn`/`damageField.splat`. `update(dt)` —
  `ejectaView.setTime`. `planetReset` — эжекта самогаснет (пул), поле чистится как сейчас.

## 5. Поток данных

1. Удар по суше → `damageField.splat` запекает профиль (чаша R + вал/эжекта A + материал G).
2. `GlobeView` сэмплит поле: вершина вдавливается/вздымается (R/A), цвет — зоны материала,
   нормаль — микрорельеф → кратер читается физически на статичном свету.
3. `ejectaView.emit` бросает баллистические частицы грунта (взлёт+падение).
4. Накопление: повторные удары сливаются `max` по каналам.

## 6. Обработка ошибок / краевые случаи

- **WebGL2-совместимость:** сохранить существующий ping-pong `max` (без `MaxEquation`), профиль
  считается опаковым выводом. Новый канал A — та же механика.
- **Полюса/шов:** `cos(lat)`-поправка и `RepeatWrapping` уже есть — вал/эжекта наследуют.
- **Перекрытие вала соседнего кратера:** `max` по A даёт наибольший вал (не сумму) — как чаша.
- **Пул эжекты исчерпан:** graceful no-op (как `ParticlePool`/`WaterBurstView`).
- **Лёд:** ветка B (полынья) сохраняется; вал/эжекта на льду — опционально приглушить (уточнить
  в плане; по умолчанию — как суша, но материал-зоны иные). YAGNI: если сложно — лёд оставить как
  есть, вал/эжекта только для суши.

## 7. Производительность

- **Профиль штампа** — разовый рендер на детонацию (не на кадр), как сейчас.
- **GlobeView фрагмент** — добавляется шум микродетали в damaged-зоне: держать дёшево (2–3 октавы),
  считать только где `(R+A)>0` (ветвление/множитель). Глобус и так фрагмент-тяжёлый — не раздувать.
- **Эжекта** — инстанс-спрайты, как гриб; десятки-сотни частиц на взрыв, один-два `InstancedMesh`.
- **Никакого динамического света** (ограничение движка) — рельеф читается статичным светом сцены.
- Цель 60fps; визуальную/перф-приёмку делает пользователь вручную.

## 8. Тестирование

Визуальную приёмку **не автоматизируем** (правило проекта) — вид проверит пользователь. Автоматом:
- **Unit (vitest, чистая логика):** профиль штампа как чистая функция (если вынесем расчёт
  bowl/rim/ejecta в тестируемый хелпер) — вал выше нуля за краем чаши, эжекта спадает к периферии,
  монотонности; баллистическая кривая эжекты (взлёт→пик→падение→0 за конечное время) на CPU-эталоне.
- **Сборка/типы:** `tsc` без ошибок; `lint` чисто.
- **Smoke:** старт без ошибок шейдеров (WebGL2).

## 9. Влияние на существующий код

- `DamageField.ts` — профиль штампа + канал A + документация каналов.
- `GlobeView.ts` — `positionNode` (вал), `colorNode` (зоны материала), нормаль-микродеталь.
- `Scene.ts` — вызов `ejectaView`.
- `config.ts` — константы (RIM_HEIGHT, радиусы зон, параметры эжекты по мощности, октавы шума).

- **Не трогаем:** `WaterBurstView`, воду (подпроект №1), sim/геймплей, LOD (2B).
