# Обломки-глыбы (этап 2 реальной разрушаемости) — дизайн

_Дата: 2026-07-14. Продолжение дорожной карты спеки `2026-07-06-voxel-crust-design.md` (§3, этап 2)._

## 1. Проблема и цель

Удар выгрызает воксели коры (этап 1), но выбитая порода просто исчезает — есть только
пылевые спрайты EjectaView. Цель этапа 2: видимый разлёт **твёрдых глыб** выбитой породы;
часть глыб выходит на орбиту и **копится в постоянное кольцо мусора** — наглядный «счётчик»
разрушения планеты и задел под этап 4 (раскол).

## 2. Согласованные решения (брейншторм 2026-07-14)

- **Орбитальные глыбы живут вечно** (до `planetReset`): постоянное кольцо, гуще с каждым
  ударом. Вытеснение самых старых при переполнении пула — естественный кап густоты.
- **Падающие глыбы — чистый визуал**: никакого вторичного урона/кратеров (YAGNI). При
  «приземлении» — маленький пылевой пух через уже готовый EjectaView.
- **Вид — инстансированные 3D-осколки**: одна низкополигональная «рваная» геометрия камня,
  разнообразие пер-инстансным неравномерным масштабом и кувырканием. Цвет — по материалу
  выбитых слоёв (грунт/порода/базальт).
- **Движение — аналитика на GPU** (паттерн EjectaView/ParticlePool): траектория — замкнутая
  формула от `uTime` в TSL, CPU пишет атрибуты один раз при взрыве, ноль работы на кадр,
  полный детерминизм. Никакой CPU-физики и никаких сущностей в ECS (глыбы не влияют на
  геймплей).

## 3. Архитектура

### 3.1. `Crust.carve` — разбивка выбитого по материалам

`CarveResult` расширяется полем `removedByMat: { soil: number; rock: number; basalt: number }`
(инкременты в существующем цикле по материалу стираемого вокселя; `MAT_WATER` не карвится).
Инвариант: `soil + rock + basalt === removed`. Из разбивки берутся и количество глыб, и
распределение их цветов (глубокий удар → больше базальта).

### 3.2. `render/DebrisView.ts` — инстанс-глыбы

Один `InstancedMesh` (низкополи-камень: икосаэдр с детерминированным джиттером вершин,
~80 треугольников) на `DEBRIS_BALLISTIC_SLOTS + DEBRIS_ORBIT_SLOTS` инстансов, кольцевые
буферы атрибутов как в EjectaView (vec4-атрибуты, DynamicDrawUsage, `flush()` по dirty).
Родитель — `globe.spinGroup` (падающие глыбы держатся своего кратера; кольцо со-вращается
с планетой — приемлемо визуально, зато один родитель).

**Слоты двумя сегментами** с независимыми кольцевыми курсорами:
- баллистический сегмент — глыбы с конечной жизнью, слоты переиспользуются;
- орбитальный сегмент — глыбы без смерти; при переполнении перезаписывается самая старая.

**Материал** — `MeshBasicNodeMaterial`, непрозрачный (никакой сортировки прозрачности;
исчезновение — схлопыванием масштаба к 0, не альфой). Псевдоосвещение — фейковый ламберт
от фиксированного направления в TSL (динамического света в проекте нет). Помним грабли
NodeMaterial: кламп выходов ≥0.

**TSL-граф позиции** (всё от `pt = uTime − spawn`, `tau = max(pt, 0)`; при `pt < 0` масштаб 0):
- Кувыркание: поворот `positionLocal`/`normalLocal` по оси `rotAxis` на угол
  `rotSpeed·tau + rotPhase` (формула Родрига), затем пер-осевой масштаб.
- **Баллистическая ветка** (как EjectaView): высота `h = max(v0·tau − ½·g·tau², 0)`,
  радиальный разлёт `rSpeed·tau` по касательной под углом `angle`;
  `life = 2·v0/g` — к приземлению масштаб плавно к 0.
- **Орбитальная ветка**: плоскость орбиты натянута на нормаль эпицентра `n` (глыба стартует
  из точки удара) и касательную `e2 = t1·cosφ + t2·sinφ` (φ — пер-инстансный азимут):
  `pos = (n·cosθ + e2·sinθ)·r`, где `θ = ω·tau` (ω со знаком — прямые и ретроградные),
  `r = mix(1 + h0, R_orb, smoothstep(0, T_asc, tau))` — спиральный взлёт с поверхности на
  орбитальный радиус `R_orb ∈ [1.25, 1.55]`. При `tau = 0` позиция = точка запуска; после
  `T_asc` — вечное кружение. Ветвление — `select` по флагу-атрибуту orbital.

Обе ветки зеркалятся чистыми CPU-функциями (`ballisticHeight` уже есть в EjectaView;
добавить экспортируемую `orbitalPos`) — юнит-тестируемы без GPU.

### 3.3. Интеграция (`Scene.startExplosion`, суша/лёд)

После `crust.carve(...)`:
`debrisView.emit(dir, yieldMt, seed, this.clock, carved.removedByMat)`.

В `emit` (детерминированный LCG от seed, как в EjectaView):
- количество: `count = clamp(round(removed · DEBRIS_PER_VOXEL), DEBRIS_MIN, DEBRIS_MAX)`;
- доля орбитальных: `DEBRIS_ORBIT_FRAC` (~0.3), остальные — баллистические;
- цвет инстанса — выбор материала пропорционально `removedByMat` (+джиттер яркости);
  soil — фиксированный бурый, rock/basalt — из `CRUST_LAYER_COLORS`;
- скорости/размеры — от мощности (`DEBRIS_SPEED_BY_YIELD`, размер ~1–2 вокселя коры);
- пылевые пыхи приземления: сразу же пишутся в EjectaView частицы с **будущим**
  `spawn = now + life_глыбы` (частица невидима до своего времени — CPU-планировщик не нужен).

`planetReset` → `debrisView.clear()` (все spawn = 1e9, курсоры в 0).

### 3.4. Конфиг (`assets/config.ts`)

`DEBRIS_BALLISTIC_SLOTS` (1024), `DEBRIS_ORBIT_SLOTS` (1536), `DEBRIS_PER_VOXEL` (~1/3),
`DEBRIS_MIN`/`DEBRIS_MAX` (8/180), `DEBRIS_ORBIT_FRAC` (0.3), `DEBRIS_SPEED_BY_YIELD`,
`DEBRIS_ORBIT_R` ([1.25, 1.55]), `DEBRIS_ASCENT_T` (сек до выхода на орбиту),
`DEBRIS_SOIL_COLOR`. Точные значения — тюнинг в реализации.

## 4. Тестирование

- **Юнит (vitest, чистый TS)**: `removedByMat` суммируется в `removed` и детерминирован;
  `orbitalPos`: старт в точке запуска (tau=0), радиус → `R_orb` после `T_asc`,
  периодичность `pos(tau) ≈ pos(tau + 2π/ω)`; баллистика: приземление при `life = 2v0/g`;
  количество глыб клампится; курсоры сегментов не пересекаются (орбитальные не затирают
  баллистический сегмент и наоборот).
- **Визуальная приёмка — вручную пользователем** (по договорённости скриншот-харнесс не
  гоняем): удар 100 Мт → разлёт глыб, часть уходит вверх и остаётся кружить; серия ударов
  уплотняет кольцо; reset очищает. Прогоняем `vitest`, `tsc`, `vite build`.

## 5. Риски

- **Перф**: ~2.5к инстансов × ~80 треугольников — мелочь для GPU, но материал один и
  дроуколл один; риска лагов по образцу этапа 1 нет (ноль CPU-работы на кадр).
- **Читаемость кольца**: со-вращение кольца со spinGroup и разброс плоскостей орбит могут
  дать «облако», а не «кольцо» — это осознанно (каждый удар даёт свою плоскость через свой
  эпицентр); густота регулируется `DEBRIS_ORBIT_SLOTS`.
- **Вода**: удары по воде глыб не дают (carve не срабатывает) — согласуется с этапом 1.
