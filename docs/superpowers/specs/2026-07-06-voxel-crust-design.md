# Воксельная кора планеты — дизайн (этап 1 «реальной разрушаемости»)

_Дата: 2026-07-06 · Статус: одобрен пользователем (мешер — Surface Nets; план — сразу)_

## 1. Проблема и цель

Текущая разрушаемость — displacement-кратеры в equirect-поле: повторные удары сливаются `max`
и НЕ накапливаются, материал «просто оставляет кратеры». Пользователь хочет реальную
разрушаемость: удары выгрызают куски коры (как обрушение части здания в играх), повторные удары
в одну точку углубляют разрушение вплоть до магмы; конечная перспектива — раскол планеты.

## 2. Согласованные решения (брейншторм 2026-07-06)

| Вопрос | Решение |
|--------|---------|
| Предел разрушения | Вплоть до раскола планеты (раскол — отдельный этап 4) |
| Путь к расколу | **Воксельная кора** (не displacement-слои) |
| Сосуществование с глобусом | **Гибрид**: нетронутая планета — текущая гладкая сфера; повреждённые регионы переключаются на воксельные чанки |
| Крупность вокселя | **~15 вокселей на диаметр кратера 100Мт** (≈40 км, стилизовано) |
| Выбитые куски | **Разлетаются** баллистически (обломки-глыбы — этап 2) |
| Мешер | **Surface Nets** (гранёные куски со скошенными краями, без лего-ступенек); изолирован в функции «чанк → геометрия», апгрейд/замена не трогает остальное |

## 3. Декомпозиция (дорожная карта)

- **Этап 0 (вне спеки, мелкие фиксы текущего поля):** кратеры меньше; урон не наползает на воду.
- **Этап 1 (этот документ):** воксельная кора — carve, слои до магмы, накопление, гибрид-рендер.
- **Этап 2:** обломки-глыбы (разлёт выбитых вокселей, часть на орбиту).
- **Этап 3:** светящиеся трещины от глубоких очагов; глобальная метрика целостности в HUD.
- **Этап 4:** раскол планеты при пороге целостности (сценарная развязка).

## 4. Архитектура этапа 1

### 4.1. `src/crust/` — состояние коры (чистый TS, без three.js)

- **Cube-sphere оболочка**: 6 граней × 256×256 столбцов × 8 слоёв вглубь. Uint8 материал-id
  на воксель, ~3 МБ. Чанки 32×32×8 → 6×8×8 = 384 шт.
- **Материалы по глубине**: `0=пусто, 1=грунт(биом), 2=порода, 3=базальт`; слои 0-1 — грунт,
  2-4 — порода, 5-7 — базальт. Под корой — магма-подложка (визуал, не воксели).
- **Океан**: столбцы с landmask=вода помечены материалом `вода` и не карвятся (заодно чинит
  «кратер наползает на воду» навсегда). Лёд — карвится как суша (полынья остаётся полем B).
- **API**:
  - `carve(dir, radiusRad, depthVox, seed)` → `{changedChunks, removedVoxels}`; carve-сфера
    центрируется на ТЕКУЩЕЙ поверхности (первый непустой воксель вдоль луча) → повторные удары
    копают глубже; лёгкий шум по краю сферы (seed) — рваные края.
  - `integrity()` → 0..1 (доля выбитого объёма; задел этапов 3-4).
  - `reset()`; детерминизм: одна последовательность carve → одно состояние (headless-тесты).

### 4.2. `render/CrustView.ts` — гибрид-рендер

- Нетронутый чанк не рендерится (0 оверхеда до первого удара).
- Удар → carve → **Surface Nets** ремешит задетые чанки (1-4 шт. по ~8К ячеек — синхронно,
  ~мс; с 1-воксельным margin соседей для нормалей/стыков).
- **Маска дырок**: equirect-текстура; регион активированного чанка помечается → GlobeView
  discard'ит там свои фрагменты; воксельный меш рисует и яму, и нетронутую поверхность чанка
  (вершины нетронутой поверхности прижаты к сфере r=1 — шов с глобусом минимален).
- **Цвет**: биом-текстура (тот же сэмпл, что глобус) на грунте + палитра слоёв (config) +
  гарь G из DamageField; дно пробития — эмиссивная **магма** (пульсация по uTime).
- **Магма-подложка**: внутренняя сфера r≈0.94 с эмиссив-шейдером, видна только сквозь пробития.
- Никакого динамического света; оба бэкенда (обычные BufferGeometry, без RT-трюков).

### 4.3. Судьба DamageField

Деформация (R-чаша, A-вал, вершинный displacement и crater-микрорельеф нормали) **удаляется** —
морфология кратера теперь честная воксельная. Поле остаётся ТОЛЬКО для перекраски: G-гарь
вокруг воронки, B-полынья льда. EjectaView/ExplosionView/DecalView-glow — без изменений.

### 4.4. Интеграция (`Scene.startExplosion`, суша/лёд)

`crust.carve(dir, R_yield, D_yield, seed)` + `damageField.splat` (только G/B) + существующие
эффекты. Радиус/глубина по мощности: 1Мт ≈ 2-4 вокселя, 10Мт ≈ 7, 100Мт ≈ 15 в поперечнике;
глубина 1Мт=1-2 слоя, 10Мт=3, 100Мт=5 (до магмы — 2-3 удара 100Мт в точку). `planetReset` →
`crust.reset()` + очистка маски дырок.

## 5. Тестирование

- **Юнит (vitest, чистая логика)**: детерминизм carve; повторный carve углубляет (копает от
  текущей поверхности); объём выбитого/integrity монотонны; океан не карвится; Surface Nets
  на эталонном чанке (снапшот числа вершин/граней, замкнутость); пробитие всех слоёв.
- **Приёмка (headless)**: серия ударов в одну точку → скриншоты прогрессии (яма глубже,
  слои темнеют, магма на дне); скол на силуэте (удар на лимбе); reset чистит.
- Перф: ремеш ≤4 чанков синхронно на удар; меши статичны между ударами.

## 6. Риски

- **Шов глобус↔чанк**: маска discard по прямоугольнику чанка + прижатие нетронутых вершин к
  сфере; допускаем стилизованный шовчик, при необходимости — margin-юбка по периметру чанка.
- **Полюса cube-sphere**: сетка граней равномернее equirect; carve через угловое расстояние —
  без особых случаев.
- **Много повреждённых чанков** (сотни мешей): three culling + при деградации перфа мёржить
  меши грани; лимит не ставим на старте.
