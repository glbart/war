# Трещины и целостность (этап 3 реальной разрушаемости) — дизайн

_Дата: 2026-07-14. Дорожная карта спеки `2026-07-06-voxel-crust-design.md` (§3, этап 3).
Решения приняты автономно по мандату пользователя (как этап 2)._

## 1. Проблема и цель

Глубокие пробития коры (до базальта/магмы) визуально ничем не отличаются от мелких, кроме
глубины ямы; накопленное разрушение планеты игроку не видно числом. Цель: **светящиеся
трещины**, расходящиеся по поверхности от глубоких очагов, и **метрика целостности коры в
HUD** — подготовка порога для этапа 4 (раскол).

## 2. Решения

- **Очаг трещин** — удар, чей carve пробил до базальта (слой ≥5 из 8). Сила очага растёт с
  глубиной пробития: слой 5 → ~0.33, слой 7 (магма) → 1.0.
- **Хранение очагов — R-канал существующего DamageField** (equirect, ping-pong max). Канал
  сейчас пишется «глубиной чаши», но НИКЕМ не читается (морфология ушла в воксельную кору) —
  переназначаем: R = интенсивность трещинного очага (штамп — спад от края кратера наружу,
  гейтится силой очага; обычный неглубокий удар пишет R=0). Ноль новых render target'ов.
- **Рисунок трещин** — процедурный в шейдере (ridged fbm по направлению фрагмента): тонкие
  «жилы», не зависящие от разрешения поля (поле только гейтит интенсивность/радиус — мыла
  на зуме нет). Свечение — эмиссивное, цвет магмы, медленная пульсация по uTime (как
  MagmaCore). Общий TSL-узел — `render/effects/cracks.ts`, подключается в **оба** материала:
  GlobeView (поверхность) и CrustView (крышки/склоны воксельных чанков) — без шва между ними.
- **integrity()** в `Crust`: `clamp(1 − removedVoxels / CRUST_DOOM_VOXELS, 0, 1)`.
  `CRUST_DOOM_VOXELS` — конфиг-константа «бюджета гибели» (не честная доля всей коры: её
  выбивать нереально долго; ~20k вокселей ≈ 34 удара по 100 Мт). 0 — порог раскола (этап 4).
- **HUD**: строка «Целостность коры: N%» в блоке статов; жёлтая <70%, красная <35%.
  Обновление — раз за кадр из main.ts (`hud.setIntegrity(scene.crustIntegrity)`), DOM
  трогается только при изменении числа. Никаких новых событий sim: целостность — метрика
  рендер-стороны (кора живёт в Scene), событийная шина не нужна.
- Никакого влияния на геймплей/жертвы (как этапы 1-2); reset чистит всё существующими путями
  (`crust.reset()` — целостность, `damageField.clear()` — очаги).

## 3. Архитектура

### 3.1. `Crust` (чистый TS)

- `CarveResult.deepestLayer: number` — самый глубокий слой d, из которого этот carve выбил
  воксель; −1, если ничего не выбито.
- `integrity(): number` — формула выше.
- `crackStrengthForDepth(deepestLayer): number` — экспортируемая чистая функция: 0 при
  слое <5, иначе `(deepestLayer − 4) / (CRUST_DEPTH_LAYERS − 5)` (слой 7 → 1), кламп [0,1].

### 3.2. `DamageField`

- Штамп: R-канал = `uCrack · oneMinus(smoothstep(0.5, CRACK_EXTENT_FRAC, dNorm))` (спад от
  ямы наружу; внутри ямы глобус всё равно discard'ится HoleMask). Каналы G/B/A — без
  изменений. Комментарий-шапка файла обновляется (R — очаги трещин).
- `splat(dir, yieldMt, kind, crack = 0)` — новый параметр силы очага (юниформ uCrack).

### 3.3. `render/effects/cracks.ts` + материалы

- `crackEmissiveNode(crackR, p, uTime)` → vec3: жилы = `smoothstep(CRACK_EDGE0, CRACK_EDGE1,
  oneMinus(abs(fbm3(p·CRACK_FREQ, 4)·2 − 1)))`, пульс = `0.78 + 0.22·sin(uTime·1.7)`,
  итог = `CRACK_COLOR · жилы · crackR · пульс` (кламп ≥0 — грабли NodeMaterial).
- GlobeView: `emissiveNode = crackEmissiveNode(dmg.r, normalize(positionLocal), uTime)`;
  новый `setTime(t)`.
- CrustView: то же с его `uvSphere`-сэмплом поля и его `positionLocal`; новый `setTime(t)`.
- Scene.update: прокидывает часы в оба (`globe.setTime`, `crustView.setTime`) — Scene уже
  владеет ссылками.

### 3.4. Scene / HUD / main

- `Scene.startExplosion` (суша/лёд): carve выполняется ДО `damageField.splat`, сила очага =
  `crackStrengthForDepth(carved.deepestLayer)` передаётся в splat.
- `Scene.crustIntegrity` (getter) → `this.crust.integrity()`.
- `Hud`: элемент `#integrity` в блоке статов; `setIntegrity(v: number)` — кэширует последний
  выведенный процент, красит порогами (70/35).
- `main.ts`: в кадровом колбэке после `scene.update(dt)` — `hud.setIntegrity(scene.crustIntegrity)`.

## 4. Тестирование

- **Юнит (vitest)**: `deepestLayer` (глубокий удар ≥5; мелкий (1 слой) <5 и ≥0; океан −1;
  детерминизм); `integrity()` (свежая кора = 1; после carve падает ровно на removed/DOOM;
  reset → 1; не уходит ниже 0); `crackStrengthForDepth` (4→0, 5→⅓, 7→1, кламп).
- **Визуальная приёмка — юзер** (по договорённости): 100 Мт по суше → из ямы расходятся
  пульсирующие светящиеся жилы; 1 Мт — без трещин; HUD-процент падает с ударами; reset
  возвращает 100% и гасит трещины. `npm test` / `npm run build` / `npm run lint` — чисто.

## 5. Риски

- **R-канал**: убедиться grep'ом, что «глубину чаши» действительно никто не сэмплит
  (GlobeView читает только G/B, CrustView — G; A не читается). Если найдётся потребитель —
  остановиться и пересмотреть (отдельное поле).
- **fbm в двух материалах** — ветка уже гоняет fbm3 (detail-слой суши); прирост стоимости
  фрагмента умеренный и гейтится нулевым R на нетронутой планете (произведение обнуляет
  вклад; сам fbm считается — как и detail).
- **Швы GlobeView/CrustView**: рисунок жил зависит только от направления фрагмента и общего
  поля — по построению совпадает на границе дискарда.
