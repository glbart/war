# Материал поверхности и разрушаемость — дизайн

_Дата: 2026-07-05 · Ветка: `feat/material-destructibility` · Статус: согласован, готов к плану_

## 1. Проблема и цель

Сейчас взрыв выглядит «кляксой на картинке»: кратер-декаль ложится поверх спутникового
фотоснимка, под которым нет никакого понятия материала. Планета не знает, где суша, а где вода,
поэтому попадание в океан выглядит так же, как по суше, и разрушать нечего.

**Цель:** ввести понятие **материала** поверхности (суша / вода / лёд + грубый биом) и на его
основе:

1. **Разрушать сушу геометрически** — воронки реально вдавливаются в поверхность и
   накапливаются.
2. **Различать воду** — попадание в океан даёт **подводный ядерный** эффект без постоянного следа
   (вода смыкается).
3. Заменить фотоснимок **стилизованной биом-раскраской** реальных материков (пустыни, льды, леса),
   оставив от карты только **слой подписей**.

Фотореализм осознанно приносится в жертву управляемости и разрушаемости. Соответствие реальному
рельефу — **примерное** (материки настоящие, биомы стилизованы правилами).

### Граница скоупа (YAGNI)

- Материал — **авторитетный факт в `sim`**, но **геймплей не меняется**: жертвы/урон считаются как
  раньше. Материал пока только (а) выбирает визуал взрыва и (б) едет в событии для будущего
  онлайна/реплеев.
- **Не** делаем: влияние биома/воды на число жертв, глубину воды (мелко/глубоко), радиацию/зиму,
  тайлы спутника при зуме, honest-соответствие реальным биомам (Кёппен и т.п.).

## 2. Согласованные решения (из брейншторма)

| Вопрос | Решение |
|--------|---------|
| Базовая картинка | **Стилизованная биом-карта**; спутниковые тайлы Esri убираем, слой подписей остаётся |
| Источник материала | **Реальный landmask** (встроенный ассет) + **биомы по правилам** (широтные пояса + шум) |
| Модель разрушения | **Геометрическая деформация** |
| Механизм деформации | **GPU displacement-поле** (накопительная damage-текстура + вершинный сдвиг) |
| Водный взрыв | **Классический подводный ядерный** (столб/купол брызг/кольцо/пар/каверна), без следа |
| Где живёт материал | **Слой в `sim`** (чистый, headless-тестируемый), **без влияния на геймплей** |

## 3. Архитектура

Шов проекта `Command → Simulation → Events → Renderer` сохраняется без изменений формы:

```
клик → DetonateCommand{dir, yield}
      → Simulation: materialAt(dir) → surface/biome
      → ExplosionStarted{dir, yield, seed, surface, biome}
      → Scene: маршрутизация
          suша/лёд → взрыв (гриб) + splat в поле урона
          вода     → WaterBurstView (подводный), в поле НЕ пишем
```

Жёсткое правило слоёв сохраняется: `sim`/`ecs` не импортируют `render`. `sim/material.ts` — чистый
TS, без three.js/canvas/network.

## 4. Слой материала (`sim`)

### 4.1. `sim/material.ts` — единственный источник истины

```ts
export type Surface = 'land' | 'water' | 'ice';
export type Biome = 'ocean' | 'ice' | 'tundra' | 'desert' | 'grass' | 'forest';

export interface Material { surface: Surface; biome: Biome; }

// Классификация точки на сфере. Чистая, детерминированная, headless.
export function materialAt(lonRad: number, latRad: number): Material;
export function materialAtDir(dir: Vec3): Material; // через dirToLonLat
```

- **Landmask.** Грубая маска суша/вода реальной Земли, упакованная в `sim/landmask.data.ts`
  (equirect ~512×256, 1 бит/пиксель ≈ 16 КБ, base64/RLE). Сгенерирована офлайн (см. 4.3),
  закоммичена — работает офлайн и детерминированно.
- **Правила биомов** (по `lat` и шуму, только для суши):
  - `|lat|` за полярным кругом → `ice` (если landmask=вода у полюса, тоже трактуем как ice-шельф) /
    `tundra` (суша чуть южнее);
  - сухие пояса `|lat|` ≈ 15–35° → `desert`;
  - экваториальная суша `|lat| < 12°` → `forest`;
  - остальное → `grass`;
  - границы поясов размываются детерминированным **шумом по `dir`** (value-noise на сид-хеше
    координат), чтобы биомы были пятнистыми, а не строго-полосатыми.
- **Surface** выводится из биома: `ocean → water`, `ice → ice`, прочее → `land`.

### 4.2. `sim/geo.ts` — обратная проекция

Добавляем `dirToLonLat(dir): { lon, lat }` (обратна существующей `lonLatToDir`):
`lat = asin(y)`, `lon = atan2(-z, x)`. Нужна и `materialAtDir`, и splat'у в рендере.

### 4.3. Офлайн-генератор ассета — `scripts/gen-landmask.*`

Node-скрипт: читает public-domain equirect-маску суша/океан (Natural Earth 1:110m ocean или
эквивалент), даунсемплит до 512×256, бинаризует (суша/вода), паковывает в `sim/landmask.data.ts`.
Запускается один раз вручную (`npm run gen:landmask`), результат коммитится. В рантайме **не**
используется. Если исходник недоступен офлайн — фолбэк: сгенерировать грубую маску из уже имеющейся
процедурной `drawProceduralEarth` или вручную обрисованных крупных материков (достаточно грубо).

### 4.4. Событие

`sim/events.ts`: вариант `explosionStarted` расширяется:

```ts
| { kind: 'explosionStarted'; id: number; dir: Vec3; yield: number; seed: number;
    surface: Surface; biome: Biome }
```

`sim/Simulation.ts` (`runMissiles`): перед `events.push(explosionStarted…)` вызывает
`materialAtDir(w.dir)` и кладёт `surface`/`biome` в событие. Больше в симуляции ничего не меняется
(жертвы считаются как прежде).

## 5. Стилизованный глобус (`render`)

### 5.1. `render/MaterialGlobe.ts` (заменяет фото-базу в `GlobeView`)

- Базовый цвет сферы строится из **того же landmask + тех же правил биомов** (единый источник
  истины с sim), запечённых в **биом-текстуру** (canvas/DataTexture) один раз при старте.
  Палитра биомов — в `assets/config.ts`.
- Поверх — процедурный шум (лёгкая вариация яркости/оттенка) для детализации, чтобы заливка не была
  плоской.
- Blue Marble как база и её bump-карта из `GlobeView` **убираются** (или репёрпоузятся под мягкий
  рельеф-нормаль по вкусу; не обязательно). Атмосфера-fresnel остаётся.

### 5.2. `render/TileLayers.ts`

- Слой `imagery` (спутник Esri) **удаляется** из массива слоёв.
- Слой `labels` (границы/названия) **остаётся как есть** (по-прежнему сетевой; офлайн → без
  подписей — текущее поведение сохраняется).

## 6. Разрушаемость: GPU displacement-поле

### 6.1. `render/DamageField.ts`

- Глобальный equirect **render-target** RGBA8, разрешение из конфига (старт: 2048×1024 ≈ 8 МБ).
- Каналы: **R = глубина воронки** [0..1], **G = гарь/копоть**, **B = оплавление/затопление** (для
  льда → полынья). A — резерв.
- **Splat при детонации** (только суша/лёд): один разовый draw (fullscreen-кусок/квад с
  ортокамерой в UV-пространстве поля) мягкого профиля «чаша + приподнятая кайма» в UV-точку
  эпицентра (`dirToLonLat → uv`), радиус по мощности (переиспользуем `ANG_PATCH_BY_YIELD`-логику).
  - **Блендинг:** `max` по R (наложения сливаются в самую глубокую воронку, не суммируются в дыру),
    `additive` (с насыщением) по G. Реализуемо двумя проходами или `MRT`; на старте допустимо
    хранить поле в одном RT и писать max через `blendEquation = MAX` для RGB с аккуратным профилем.
  - **Искажение equirect:** U-протяжённость splat масштабируется `1/cos(lat)`; по шву `lon=±180`
    делаем wrap (при близости к краю — второй splat со сдвигом на ширину). Приближение принимаем.
- **Сброс** (`clear()`): очистка RT в ноль (вызывается на `planetReset`).

### 6.2. Материал глобуса (TSL, оба бэкенда)

- **vertexNode:** сэмпл `damage.R` по UV вершины → сдвиг позиции вдоль нормали внутрь на
  `depth * MAX_DEPTH`, плюс приподнятая кайма из профиля. Требует плотной сетки: сфера
  `SphereGeometry(1, LON_SEG, LAT_SEG)` со значениями из конфига (старт: 384×192 ≈ одна статичная
  сетка, один draw — дёшево). Vertex-texture-fetch поддержан и в WebGPU, и в WebGL2.
- **colorNode:** биом-цвет мешается к гари (тёмная копоть) по G и к остекловке/оплавлению по B; на
  дне воронки — мягкое затенение по градиенту глубины (дешёвый псевдо-AO). Раскалённая свежая кайма
  — короткоживущий член (см. 6.3).

### 6.3. Судьба текущего `DecalView`

- Постоянный кратер-декаль **вытесняется** полем урона (`craterMesh` больше не нужен).
- Остывающая горячая кайма (`glow`, hot→cold за 50 с) **сохраняется** как транзиентный эффект:
  либо остаётся отдельным лёгким additive-мешем (минимальная переделка), либо переносится в член
  поля с временным затуханием. На старте — оставить существующий glow-механизм, убрать только
  постоянный кратер. `DecalView` ужимается/переименовывается в `HeatGlowView` (или его функции
  переезжают).

## 7. Взрывы по материалу (`render`)

`Scene.startExplosion` получает `surface`/`biome` и маршрутизирует:

- **`land`:** текущий `ExplosionView` (огненный шар + ударная волна) + `ParticlePool` (гриб) +
  splat в поле урона + горячая кайма. Цвет пыли гриба тонируется по биому (песок/гарь/снег) — через
  uniform в `ParticlePool.emit`.
- **`water`:** новый **`render/WaterBurstView.ts`** (пул, как `ExplosionView`): купол брызг
  (расширяющаяся белая полусфера) + вертикальный столб (вытянутый белый плюм) + расходящееся пенное
  кольцо по поверхности (транзиентная декаль) + облако пара + кратковременная каверна, которая
  смыкается. **В поле урона не пишем.** Огненный шар суши не используется.
- **`ice`:** воронка во льду (splat как суша) + пометка `B` (оплавление) → центр в colorNode
  рендерится открытой водой (полынья) с каймой битого льда. Если окажется дорого/сложно — деградация
  до обычной воронки суши (флаг в конфиге).

## 9. Перф и совместимость

- Глобус — **одна** плотная статичная сетка (384×192), один draw. Поле урона — одна текстура; splat
  = **1 draw на взрыв** (события редки), не на кадр. В кадре добавляется +1 vertex-fetch и пара
  фрагментных сэмплов — пренебрежимо.
- Правило **«никакого динамического света»** сохраняется (вспышка/кайма — additive).
- Vertex-texture-fetch и RTT работают и в **WebGPU**, и в **WebGL2** → паритет бэкендов сохраняется;
  весь новый шейдинг — на **TSL**, без сырого GLSL/WGSL.
- Цель стресс-теста (среднее время кадра **< 20 мс**) остаётся; ожидаем прежние ~2 мс.

## 10. Тестирование

- **Sim (Vitest, headless):**
  - `materialAt` на реперных точках: Сахара→`desert`, Антарктида→`ice`, центр Тихого→`water`,
    Амазония→`forest`, Гренландия→`ice`; санити береговой линии (материки не в океане); детерминизм
    (одинаковый вход → одинаковый выход).
  - `dirToLonLat` ∘ `lonLatToDir` = identity (в пределах эпсилон).
  - Пламбинг: детонация над водной точкой даёт событие с `surface='water'`.
- **Render (headless Chrome + скриншот):** биом-глобус; видимая вмятина кратера на лимбе
  (силуэт); накопление/слияние двух близких воронок; водный столб/кольцо; полынья во льду; `reset`
  очищает поле; оба бэкенда (WebGPU + WebGL2 swiftshader) без ошибок консоли; стресс-тест 15 ударов
  держит < 20 мс.

## 11. Риски и открытые вопросы

- **Ассет landmask** — главная внешняя зависимость подготовки. Митигируется офлайн-скриптом из
  public-domain источника; при недоступности — грубая ручная/процедурная маска (материки настоящие,
  береговая линия примерная — этого достаточно).
- **Искажение splat у полюсов и на шве** — принимаем приближение (`1/cos(lat)` + wrap).
- **`max`-блендинг глубины в одном RT** — если окажется неудобным на одном из бэкендов, фолбэк:
  двойной RT ping-pong (читаем прошлое поле, пишем `max` в новое). Уточняется на реализации.
- **Резкость кратеров при сильном зуме** ограничена `DAMAGE_TEX_SIZE`/тесселяцией. Т.к. спутник
  убран, близкий план всё равно стилизован — приемлемо; при нужде поднять разрешение поля.
- **Обновление банка памяти** после реализации: `activeContext.md`/`progress.md`
  (+ `systemPatterns.md` — новый паттерн «поле урона»).
