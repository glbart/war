# Баллистические МБР и залп — дизайн

_Дата: 2026-07-14. Решения приняты автономно по мандату пользователя._

## 1. Проблема и цель

Сейчас единственный способ удара — клик: боеголовка прилетает «из космоса» по радиусу.
Цель: **ракеты летят как реальные МБР** — старт с поверхности, баллистическая дуга
(разгон → свободный полёт с апогеем → быстрый вход), и **кнопка залпа**: из нескольких
случайных точек суши ракеты летят в другие места. Ручной клик («удар из космоса») остаётся.

## 2. Решения

- **Траектория** — стилизованная баллистика (без интеграции гравитации): путь =
  `slerp(from, to, e) · (1 + apex·sin(π·e))`, где `apex` растёт с дальностью
  (межконтинентальная дуга выше), `e = k^1.6` — медленный разгон со старта, быстрый вход
  в атмосферу (реальный профиль МБР: буст медленный, реентри быстрый). Время полёта тоже
  от дальности (3–8 с). Чистый модуль `sim/ballistics.ts` (slerp, apexFor, flightTimeFor,
  ballisticPos) — юнит-тесты, общий для sim (тайминги) и рендера (позиция каждый кадр).
- **Sim авторитетен**: новая команда `{ kind: 'salvo' }`. Симуляция детерминированно (Rng)
  выбирает `SALVO_COUNT` стартовых точек **на суше** (rejection sampling по materialAtDir,
  фолбэк — любая точка) и целей — **случайных живых городов** (жертвы и лента работают);
  если живых городов нет — случайные точки. Мощность залпа — текущая выбранная (`setYield`).
- **Warhead** получает опциональные `from?: Vec3` и индивидуальный `flightTime` (для
  баллистических — от дальности; «из космоса» — прежние 2.6 с). Событие `missileLaunched`
  расширяется `from?` и `flightTime` — рендер больше не хардкодит длительность (текущий
  дубль констант 2.6 в Simulation и MissileView устраняется).
- **Рендер** (`MissileView`): слот в двух режимах — 'space' (прежняя радиальная дуга) и
  'ballistic' (позиция из `ballisticPos`, нос по касательной — lookAt на точку чуть впереди).
  Пул/деспаун/фолбэк-таймер — как есть.
- **HUD**: кнопка «☢ Залп МБР» под кнопками мощности → `host.post({ kind: 'salvo' })`.
- Взрыв/жертвы/кора — без изменений (детонация в `to`, как раньше в `dir`).

## 3. Тестирование

- **Юнит**: ballistics — `ballisticPos(from,to,0)=from` и `(…,1)=to` (радиус 1), апогей в
  середине ≈ `1+apex`, slerp остаётся единичным, монотонность `e(k)`; flightTimeFor растёт
  с дальностью. Simulation — `salvo` порождает SALVO_COUNT missileLaunched с `from` на суше
  и целями-городами, детерминизм от seed, взрывы приходят по индивидуальным flightTime.
- **Визуал — юзер**: клик — прежний удар из космоса; кнопка залпа — дуги от поверхности
  к городам, нос по траектории. Прогоны: `npm test` / `npm run build` / `npm run lint`.

## 4. Конфиг

`SALVO_COUNT = 6`; `BALLISTIC_APEX_MIN = 0.12`, `BALLISTIC_APEX_SCALE = 0.5` (апогей =
MIN + SCALE·ang/π, в радиусах планеты); `BALLISTIC_TIME_MIN = 3`, `BALLISTIC_TIME_SCALE = 5`
(сек, время = MIN + SCALE·ang/π); `BALLISTIC_EASE_POW = 1.6`.

## 5. Риски

- Пул MissileView (16 слотов) при залпе 6 + ручной спам — достаточно; graceful no-op остаётся.
- Города как цели после гибели всех — фолбэк на случайные точки (кнопка не «ломается»).
- `missileLaunched` без `from` (старые пути) → режим 'space' — обратная совместимость.
