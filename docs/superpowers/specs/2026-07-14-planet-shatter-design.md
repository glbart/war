# Раскол планеты (этап 4, финал реальной разрушаемости) — дизайн

_Дата: 2026-07-14. Дорожная карта спеки `2026-07-06-voxel-crust-design.md` (§3, этап 4).
Решения приняты автономно по мандату пользователя (как этапы 2-3)._

## 1. Проблема и цель

Целостность коры (этап 3) доходит до 0% — и ничего не происходит. Цель: **сценарная
развязка** — планета, добитая до нулевой целостности, раскалывается: агония с разгорающимися
трещинами по всей поверхности → распад на рой крупных осколков вокруг обнажённого
магматического ядра. Финал песочницы; «Восстановить планету» отменяет всё.

## 2. Решения

- **Машина состояний рендера**: `intact → agony → shattered`, живёт в Scene (кора и
  integrity — рендер-сторона, sim не знает и не должен: раскол не влияет на подсчёты sim).
  Чистая логика фаз — отдельный модуль `render/shatterState.ts` (тестируется headless).
- **Триггер**: сразу после carve очередного удара `crust.integrity() === 0` (и состояние
  intact) → агония.
- **Агония** (`SHATTER_AGONY_T` ≈ 4.5 с): глобальный буст трещин 0→1 (добавка к R-каналу в
  общем узле cracks.ts — жилы разгораются по ВСЕЙ планете, не только у очагов), нарастающая
  тряска камеры, низкий грохот (`playShatter`). Удары в агонии продолжают приниматься.
- **Раскол** (по истечении агонии): скрываются глобус, атмосфера, океан-оболочка, воксельные
  чанки и слой границ/подписей; магма-ядро (r=0.945) остаётся и разгорается (буст цвета);
  `DebrisView.emitShatter` — ~140 КРУПНЫХ вечных осколков (орбитальный сегмент: радиусы
  1.05–1.9, медленное кружение/кувыркание, размеры 0.05–0.18, цвета порода/базальт/грунт) —
  «поле обломков» вокруг ядра вместе с накопленным кольцом этапа 2; второй грохот громче;
  мощный толчок тряски; HUD-баннер «☠ ПЛАНЕТА РАСКОЛОТА».
- **В состоянии shattered** события `missileLaunched`/`explosionStarted` игнорируются Scene
  (планеты нет; рейкаст PointerController попадает и по невидимому мешу — глушим на
  приёмнике). Sim продолжает жить своей жизнью — его счётчики не сценарий этапа.
- **Reset**: `planetReset` возвращает всё (состояние intact, буст 0, видимости, баннер;
  поля/кора/глыбы чистятся существующими путями).

## 3. Архитектура

### 3.1. `render/shatterState.ts` (чистый TS)

`class ShatterState`: `phase: 'intact' | 'agony' | 'shattered'`; `boost: number` (0..1 —
глобальный буст трещин, в агонии растёт линейно до 1, в shattered = 1); `trigger()` —
intact→agony; `update(dt): 'shatter' | null` — тикает агонию, возвращает `'shatter'` ровно
один раз в момент перехода agony→shattered; `reset()`. Конфиг: `SHATTER_AGONY_T`.

### 3.2. Визуальные крючки

- `cracks.ts`: `crackEmissiveNode(crackR, p, uTime, boost)` — 4-й аргумент-узел,
  `effCrack = clamp(crackR + boost, 0, 1)`.
- `GlobeView`: юниформ `uCrackBoost` + `setCrackBoost(v)`; `setPlanetVisible(v)` — глобус +
  атмосфера (сохранить ссылку на atmo-меш).
- `CrustView`: юниформ `uCrackBoost` + `setCrackBoost(v)`; `setVisible(v)` — вся группа чанков.
- `MagmaCore`: юниформ `uBoost` + `setBoost(v)` — цвет к бело-жёлтому и ярче.
- `DebrisView.emitShatter(seed, now)`: пишет `SHATTER_SHARD_COUNT` крупных орбитальных глыб
  (реюз writeDebris/слотов; вечные — переживают всё до reset).
- `sound.ts`: `playShatter(intensity)` — длинный (10 с) низкий грохот по образцу playBoom
  (шум с ФНЧ 300→18 Гц + тон 34→14 Гц).
- `Hud.setShattered(v)`: баннер `#shatter` («☠ ПЛАНЕТА РАСКОЛОТА») над статами; кэш как у
  setIntegrity.

### 3.3. Scene / main

- Scene: поле `shatter = new ShatterState()`. После carve в `startExplosion`: если
  `integrity()===0` → `shatter.trigger()` + первый `playShatter(0.7)`. В `update(dt)`:
  `shatter.update(dt)`; пока агония — `rig.shake = max(rig.shake, boost·0.05)`; буст в
  GlobeView/CrustView/MagmaCore; если вернулось `'shatter'` — скрыть планету (globe,
  oceanShell.mesh, crustView), `debrisView.emitShatter`, `playShatter(1.6)`, толчок тряски.
- `handleEvent`: в shattered игнорировать `missileLaunched`/`explosionStarted`; в
  `planetReset` — `shatter.reset()` + вернуть видимости/бусты/баннер.
- Геттер `isShattered` для main/Hud.
- main.ts: раз в кадр `hud.setShattered(scene.isShattered)` и
  `tiles.group.visible = !scene.isShattered`.

## 4. Тестирование

- **Юнит (vitest)**: ShatterState — intact не тикает; trigger→agony; boost растёт 0→1 за
  AGONY_T (линейно, кламп); `'shatter'` возвращается ровно один раз; после — phase
  shattered, boost=1; reset из любой фазы → intact/0; повторный trigger в agony/shattered —
  no-op.
- **Визуальная приёмка — юзер** (по договорённости): добить планету до 0% (можно временно
  снизить CRUST_DOOM_VOXELS) → агония с разгорающимися жилами и тряской → распад: ядро +
  рой осколков, баннер; удары не работают; reset возвращает планету. `npm test` /
  `npm run build` / `npm run lint` — чисто.

## 5. Ревизия 2026-07-14 (фидбэк пользователя): киношный разрыв на куски

Мгновенное «планета исчезла → рой мелких осколков» заменяется настоящим разрывом:
- **Куски-плиты**: сферическая оболочка (внешний радиус 1, внутренний чуть выше магмы)
  разбивается на `SHATTER_PLATE_COUNT` (~14) замкнутых кусков — Вороной по сфере на
  икосфере (детерминированно от seed): внешняя поверхность + внутренняя + стенки по границам
  регионов. Модуль геометрии — чистый TS (`render/shatterShards.ts`), тестируется
  (замкнутость кусков: каждое ребро встречается чётное число раз; сумма внешних треугольников
  = треугольники икосферы; детерминизм).
- **Бесшовная подмена**: в момент 'shatter' куски вместе составляют ровно ту же сферу —
  глобус скрывается, куски появляются, силуэт не меняется.
- **Замедленный разлёт («как в фильме»)**: каждый кусок уплывает вдоль направления своего
  центра со скоростью, нарастающей от нуля (`smoothstep` за `SHATTER_PLATE_RAMP_T` ≈ 6 с),
  с медленным кувырканием. Никакого мгновенного скачка — сначала расходятся щели, сквозь
  них светится ядро, потом куски разгоняются в стороны. Движение — CPU-трансформы
  ~14 мешей на кадр (дёшево, без аллокаций).
- **Материал кусков**: внешняя корка — биом-текстура (equirect-uv из локальной позиции, как
  CrustView — локальные координаты куска не меняются при полёте); срезы/изнанка — градиент
  порода→магма по глубине (1−r), с магмовой эмиссией (изнутри куски раскалены);
  side=DoubleSide (стенки видны с любой стороны).
- Рой мелких осколков (`emitShatter`) остаётся как сопутствующий мусор; reset убирает куски
  (dispose) и возвращает глобус.

## 6. Ревизия 2026-07-14 №2 (фидбэк): от планеты не остаётся ничего

Реальный взрыв планеты не оставляет ядра. Продолжение сценария после 'shatter':
- Через `SHATTER_CORE_LINGER_T` (~7 с; куски успевают отойти и обнажить ядро) — **распад
  ядра**: слепящая вспышка (магма-сфера коротко раздувается и разгорается до пересвета),
  под её прикрытием орбитальное кольцо мусора очищается и заменяется **разлётом прочь**
  (`DebrisView.emitEscape`: ~200 обломков на «орбиты» радиусом 6–14 — за секунды уходят
  из вида и практически исчезают), затем магма схлопывается в ноль и скрывается; финальный
  грохот. Фазы: `shattered → collapse (прогресс 0→1 за SHATTER_CORE_COLLAPSE_T) → gone`.
- Итог: пустота со звёздами, вдаль улетают куски-плиты (навсегда, ~14 мешей — дёшево),
  баннер «планета расколота» остаётся. Reset возвращает всё (магма: scale/visible/boost).
- ShatterState расширяется фазами `collapse`/`gone` и полем `coreProgress` (0→1 в collapse,
  1 в gone); `update` дополнительно возвращает `'collapse'` ровно один раз.

## 7. Риски

- **Слоты осколков**: emitShatter пишет в орбитальный сегмент (1536) — 140 крупных + копившееся
  кольцо помещаются с запасом; вытеснение старейших допустимо.
- **Рейкаст по невидимому глобусу**: three не учитывает visible в raycast — потому глушим
  события на приёмнике Scene (не в PointerController: sim-команды продолжают работать для
  reset-кнопки, менять ввод не нужно).
- **Tiles/подписи**: скрываются целиком через `tiles.group.visible` из main — не трогаем
  внутренности TileLayers.
