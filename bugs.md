# Разбор багов — ИТОГ (2026-07-06: все баги из этого файла исправлены)

Ветка: `fix/ocean-mask-flip`. Ниже — найденные корневые причины и фиксы. Проверено:
`npm test` (67 зелёных), `npm run build`, `npm run lint`, `npm run accept` (кратер/полынья
локальны и в точке удара; вода даёт каверну+пену+волны в точке удара; после reset чисто).

## 1. РЕШЁН: удар по суше красил ВСЮ сферу в серое зерно

**Корень:** opaque-путь node-материалов three. При `transparent=false` + `NormalBlending`
`NodeBuilder.isOpaque()` → `NodeMaterial.setupDiffuseColor` принудительно пишет
`diffuseColor.a = 1.0`. Штамп `DamageField` — фуллскрин-квад, а канал A = вал+эжекта →
КАЖДЫЙ splat заливал A=1 по всей текстуре → `craterMask≈1` по всей сфере → серый fbm-микрорельеф
и «пыльная» перекраска всей планеты. Вода не серила, потому что не зовёт `DamageField.splat`.

**Фикс:** `mat.blending = THREE.NoBlending` у материала штампа (`src/render/DamageField.ts`) —
выводит материал из-под opaque-условия, альфа доходит до RT как данные.

## 2.1 РЕШЁН: удар по воде «зеркалился» вторым очагом

**Корень:** V-конвенция сэмплинга render-target-текстур в three (оба бэкенда):
`texture(rtTex, uv)` при `uv.y=q` возвращает фрагмент, отрисованный в квад-координате `y=1−q`
(на GL — авто-flipY для `isRenderTargetTexture` в `TextureNode.setupUV`; на WebGPU — top-left
origin). Шаг симуляции `WaterField` читал СВОЙ ЖЕ RT прямым `uv()` → каждый шаг зеркалил всё
поле по широте → на 60 fps рябь мерцала между точкой удара и зеркальной широтой = «два очага».

**Фикс:** самочтение в координатах записи — `uvR = (x, 1−y)` в `buildStepMaterial`
(`src/render/WaterField.ts`); тот же латентный баг закрыт в `DamageField.prevSample`
(зеркалирование накопленного урона при повторных splat). Формулы записи splat
(`v=(π/2−lat)/π`) — ПРАВИЛЬНЫЕ, их не трогать: они уже учитывают отражённый RT-сэмплинг
(проверено скриншотами: кратер 23°N и каверна 40°N ложатся ровно в точку удара).

## 2.2 РЕШЁН: «белый круг» при ударе по воде + (новый) «удар в пустоту»

Импульс теперь бьёт воду ВНИЗ (каверна, `.negate()` в `WaterField`), пена — только на
положительных гребнях (`strikeFoam = smoothstep(0.12, 0.5, fieldH)` в `OceanShell`).

**Критичный корень по пути:** стандартный вывод node-материалов КЛАМПИТСЯ в ≥0
(`NodeMaterial.setup`: «force unsigned floats» — `vec4(...).max(0)`), поэтому отрицательные
высота/скорость волны молча обнулялись при записи — с отрицательным импульсом поле оставалось
тождественным нулём (юзер-репорт «даже волны нет»), а раньше волны были полувыпрямленными.
**Фикс:** запись через `mat.outputNode` (минует кламп и tone-mapping) + `NoBlending`.

## 2.3 Тюнинг волн (проверить визуально вживую)

- `WATER_WAVE_SPEED` 0.25 → 0.1 (волны заметно медленнее расходятся);
- `WATER_HEIGHT_DAMPING` 0.02 → 0.003 (кольца живут секунды и успевают разойтись);
- `WATER_SPLAT_STRENGTH` → {1: 0.9, 10: 1.6, 100: 2.6} (не выше ~2.6 — иначе кламп ±4
  делает плоское «блюдо» вместо каверны);
- амплитуда отклика в `OceanShell`: `fieldH·MAX_CRATER_DEPTH·3.0` (было 1.5).
